netmon — сетевой монитор в стиле htop
=======================================

Назначение
----------
Консольное приложение, отображающее в реальном времени:
* список слушающих TCP/UDP-серверов с PID, процессом, трафиком и количеством клиентов;
* детализацию клиентов (включая подробную информацию об OpenVPN‑клиентах);
* сводку по активным пользователям, сессиям SSH и неудачным попыткам входа.

Требования
----------
* Linux
* Python 3.10+
* пакет psutil
* библиотека textual (Textual)
* права root (или sudo) для доступа к сетевым соединениям и /proc

Установка
---------
1. Склонируйте или скопируйте каталог netmon/ в нужное место.
2. Установите зависимости:
   pip install psutil textual
3. Для работы с логами входа (журнал sshd) желательно, чтобы текущий
   пользователь мог читать /var/log/auth.log или выполнять journalctl.
   При необходимости используется sudo -n (без пароля).

Запуск
------
От root или с sudo:
   sudo python -m netmon.netmon

или если создан __main__.py:
   sudo python -m netmon

Для удобства можно сделать скрипт запуска, например:
   #!/bin/bash
   cd /путь/к/netmon
   source venv/bin/activate   # если используется venv
   sudo -E python -m netmon

Горячие клавиши
---------------
↑ / ↓              – навигация по таблице серверов
Enter              – выбрать сервер, показать его клиентов
/                  – поиск/фильтр по PID, имени или порту
ESC                – сбросить фильтр
Пробел             – пауза / возобновление автообновления (2 сек.)
S                  – показать / скрыть таблицу клиентов
?                  – справка
Q                  – выход

Особенности
-----------
* Для OpenVPN‑серверов приложение автоматически подключается к
  management‑интерфейсу (TCP, по умолчанию 127.0.0.1:5555) и
  получает детальный список клиентов: Common Name, реальный адрес,
  принято/отправлено байт, время подключения.
* Адрес management‑интерфейса определяется из аргументов командной
  строки процесса или из конфигурационного файла OpenVPN.
* Если management‑интерфейс защищён паролем, клиенты не отображаются.
* Права root необходимы для чтения /proc/<pid>/io, /proc/<pid>/cgroup
  и списка сетевых соединений. Без них монитор покажет сообщение
  "No permission".

Зависимости модулей:
  netmon.py использует коллекторы из collectors.py и монитор OpenVPN,
  а также help_screen.py для вызова справки.

Примечания по интерфейсу
------------------------
* Таблица серверов полностью перестраивается каждые 2 секунды (как в htop),
  но позиция курсора и прокрутка сохраняются.
* В столбце PROCESS дополнительно показывается имя systemd‑сервиса
  (если процесс запущен через systemd), например:
  openvpn (openvpn-server@server.service).
* Трафик раскрашивается: зелёный — низкий, жёлтый — средний, красный — высокий.
* В таблице клиентов OpenVPN отображаются отдельные колонки с именами,
  адресами и трафиком; для остальных серверов — адрес, состояние и PID клиента.
